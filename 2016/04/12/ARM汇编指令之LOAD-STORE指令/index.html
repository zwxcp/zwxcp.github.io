ARM汇编指令之LOAD-STORE指令

发表于 2016-04-12   |   分类于 ARM嵌入式编程   |  

load-store指令用于在存储器和处理器寄存器之间传输数据。共有3中类型的load-store指令：单寄存器传输指令、多寄存器传输指令和交换指令。

1. 单寄存器传送指令

该类指令用于把单一的数据传入或者传出一个寄存器。支持的数据类型有字（32位）、半字（16位）和字节。

;读数据到r0
LDR r0, [r1]        ; = LDR r0, [r1, #0] 从一个以寄存器r1内容为地址的存储器中装载数据。
;存储数据
STR r0, [r1]        ; = STR r0, [r1, #0]

第一条指令从存放在r1中的存储器地址读取一个字，让后存放到r0中；第二天指令把r0的值写入寄存器r1所指向的地址中。从寄存器r1的偏移量为0，寄存器r1被称为基地址寄存器。

2. 单寄存器laod-store指令的寻址方式

变址模式组合：回写前变址(preindex with writeback)、前变址(preindex)及后变址(postindex).

  • 回写前变址：在计算出新的地址后要用新的地址更新基址寄存器的内容，然后再利用新的基址寄存器进行寻址；
  • 前变址：在计算出新的地址后要用新的地址更新基址寄存器的内容，基址寄存器在操作之后仍然保持原值；

  • 后变址：先使用基址寄存器的原值进行寻址操作，操作完成后再更新基址寄存器。

    ;操作前：
r0 = 0x00000000
r1 = 0x00009000
mem32[0x00000000] = 0x01010101
mem32[0x00009004] = 0x02020202

;回写型前变址寻址:
LDR r0, [r1, #4]！        ;取r1 + 4地址上的值，赋值给寄存器r0，再将r1寄存器值+0x04，r1 = r1 + 4
;结果：
r0 = 0x02020202
r1 = 0x00009004

;前变址寻址：
LDR r0, [r1, #4]        ;取r1 + 4地址上的值，赋值给寄存器r0，r1寄存器值不变
;结果：
r0 = 0x02020202
r1 = 0x00009000

;后变址寻址：
LDR r0, [r1], #4        ；取r1地址上的值，将值赋给寄存器r0，再将r1寄存器值+0x04，r1 = r1 + 4
;结果：
r0 = 0x01010101
r1 = 0x00009004


有符号的偏移量或寄存器用“+/-”表示，表示相对一个基址寄存器Rn的正偏移量或负偏移量。基址寄存器是一个指向字节的指针，偏移量表示偏移的字节数。
寻址方式中的立即数说明地址是通过基址寄存器和一个编码再指令中的12位偏移量计算而得；寄存器说明地址是通过基址寄存器和另一个特定的寄存器中的内容计算而得；

3. 多寄存器传送指令

多寄存器传送指令可以用一条指令传送多个寄存器的值到内存，或者从内存数据到多个寄存器。传输是从一个指向存储器基地址寄存器Rn开始的。多寄存器传送指令在数据块操作、上下文切换、堆栈操作等方面，比单寄存器传送指令会有更高的执行效率。
但多寄存器的load-store指令会增加中断的延迟，因为ARM通常不会打断正在执行的指令去响应中断，而必须等待指令执行完成。
语法：

{} <寻址模式> Rn{!}, {r^}

堆栈操作：

ARM使用多寄存器的load-store指令完成堆栈操作。pop操作（出栈）使用一条多寄存器的load指令，push操作（入栈）使用一条多寄存器的store指令。在ATPCS中，堆栈被定义为递减式满堆栈，LDMFD和STMFD指令分别用来支持pop和push功能。
STMFD指令把寄存器内容放入堆栈，并更新sp的值；

4. 交互指令

交换指令是load-store指令的一种特例，它把一个存储单元的内容与寄存器内容相交换。交换指令在执行期间不能被其它任何指令或其它任何总线访问打断，在此期间系统“占据总线”，直至交换完成。
交互指令多用于实现操作系统中的信号量和互斥操作。
